This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

sumi is a TUI-first Hyprland rice for Arch Linux with hardware-agnostic support. The repo contains a Go CLI (Bubble Tea TUIs) that handles the full lifecycle — bootstrap, install, update, uninstall, theme, config, doctor, status, cleanup, control, welcome — plus the dotfiles themselves. Configs are symlinked from the repo into `~/.config/`.

## Build commands

```bash
make build      # go build -o sumi ./cmd/sumi
make static     # CGO_ENABLED=0 static binary (for Arch ISO)
make install    # build + install to ~/.local/bin/sumi
make test       # go test ./...
make cover      # test with coverage report
make clean      # remove ./sumi binary
```

On Windows (for development only): `go build -buildvcs=false -o sumi ./cmd/sumi`

Unit tests cover config, hardware detection, migrations, model types, and checkpoint logic. Run `go test ./...` to verify.

## Go architecture

**Entry:** `cmd/sumi/` — cobra CLI with subcommands split across files (main.go, install.go, bootstrap.go, update.go, uninstall.go, theme.go, config.go, doctor.go, status.go, cleanup.go, control.go, welcome.go, helpers.go).

**`internal/model/step.go`** — shared types breaking the ui ↔ steps import cycle. `Step` struct (ID, Section, Name, RunGo, RunStream, Skip, Status), `StepStatus`, `InstallCtx`, `GoResultMsg`, `NextStepMsg`.

**`internal/ui/`** — Bubble Tea models:
- `installer.go` — install TUI with checkpoint resume and retry/skip on failure
- `steprunner.go` — generic reusable step runner (used by doctor, uninstall)
- `bootstrap.go` — two-phase bootstrap: huh forms (config) → step runner (execution)
- `config.go` — huh form for editing ~/.config/sumi/config.toml
- `theme.go` — huh form for wallpaper/theme management (native toggle, no shell)
- `status.go` — lipgloss render of system state
- `control.go` — Bubble Tea list control center (replaces fzf script)
- `welcome.go` — multi-page first-run wizard
- `progress.go` — progress bar renderer
- `common.go` — shared helpers (RenderSteps, RunCurrentStep, NewSpinner, NewViewport)

**`internal/runner/shell.go`** — `Stream` type for async subprocess output. Key functions: `RunCmd`, `RunCmdDir`, `RunCmdWithStdin` (LUKS passwords), `WriteAsSudo`, `ReadFile`, `Func` (wraps Go func as Stream).

**`internal/steps/`** — step implementations for install, doctor, uninstall. Each file exports functions returning `[]model.Step`. `errors.go` provides `DiagnoseError` and `DoctorFix` for actionable error messages.

**`internal/checkpoint/`** — install progress tracking. Checkpoint file at `~/.cache/sumi/install-progress.json`. Supports resume-from-failure, step hashing for incremental updates.

**`internal/bootstrap/`** — Arch ISO installation logic: network, disk, partition, luks, filesystem, pacstrap, chroot, staging, errorpage. `steps.go` wires them into `[]model.Step`.

**`internal/hardware/`** — DMI-based hardware detection via `DMIReader` interface. `Profile` struct with Packages, AURPackages, KernelParams, Modules, Services, Tweaks. Profiles: Framework 13 AMD, ThinkPad, ASUS ROG, GenericLaptop, GenericDesktop.

**`internal/config/`** — TOML config at ~/.config/sumi/config.toml. Sections: battery, theme, cleanup, hardware. `LoadFrom`/`SaveTo` for testability.

**`internal/theme/toggle.go`** — native Go dark/light theme toggle (writes Hyprland color vars, no shell scripts).

**`internal/theme/palette.go`** — monochrome identity palette (no Catppuccin). Base grays + semantic colors (Green, Red, Yellow, Blue). Accent comes from wallpaper at runtime.

**Dependencies:** bubbletea + bubbles + lipgloss (TUI), huh (forms), cobra (CLI), BurntSushi/toml (config). Module is `sumi`, Go 1.22.

## Config & dotfiles structure

- `hypr/` — Hyprland config. `hyprland.conf` sources `conf.d/` which contains `keybinds.conf`, `rules.conf`, `autostart.conf`, `env.conf`, and `colors.conf` (generated by wallust — never edit manually)
- `scripts/` — bash scripts called by Hyprland keybinds (wallpaper, screenshots, control center, etc.)
- `wallust/` — color extraction config + templates that generate color files for foot, waybar, fuzzel, dunst, hyprlock, cava, btop, tmux
- `systemd/user/` — user services (cliphist, wallust-watcher, lock-before-sleep, cleanup timer)
- `migrations/` — numbered shell scripts (`NNNN-name.sh`) applied once; tracked in `~/.local/share/sumi/applied-migrations`

## Wallpaper/color pipeline

`sumi wallpaper apply <image>` → hyprpaper IPC → wallust generates colors → restarts waybar/dunst. A lockfile at `~/.cache/sumi/.wallpaper-applying` prevents inotify feedback loops. `wallust-watcher.service` watches `~/.cache/sumi/current-wallpaper`.

**Critical:** Color output files (`foot/colors.ini`, `waybar/colors.css`, `fuzzel/colors.ini`) are **copied** on install, not symlinked. Wallust overwrites them at runtime. Never symlink color files.

## Coding conventions

### Go
- Standard `gofmt` formatting (tabs)
- `//nolint:errcheck` and `//nolint:gosec` used inline for known-safe suppressions
- Error wrapping: `fmt.Errorf("context: %w", err)`
- No global state; `model.InstallCtx` passed to step functions
- `SUMI_DIR` env var overrides repo root detection when binary is symlinked
- Import cycle prevention: shared types live in `internal/model/`

### Shell scripts
- `#!/usr/bin/env bash` with `set -euo pipefail`
- Box-style header comments
- Should pass `shellcheck`

### Indentation (from .editorconfig)
- Tabs: `*.go`, `Makefile`
- 4 spaces: `*.sh`, `*.lua`, `*.conf`, `*.ini`, `*.cfg`, `*.py`
- 2 spaces: `*.json`, `*.jsonc`, `*.css`, `*.yaml`, `*.toml`
- LF line endings enforced globally

## Hyprland version notes (0.50+)

- Gestures: `workspace_swipe*` removed → `gesture = 3, horizontal, workspace`
